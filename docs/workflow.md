# ワークフロー — 4プロンプト構造

ネコネコ SNS Content Engine のコンテンツ制作は、4 つ（+セットアップ）のプロンプトを順に流すパイプラインで構成します。各フェーズは前フェーズの出力を入力として受け取ります。

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 00 SETUP │──▶│ 01 NICHE │──▶│ 02 SCRIPT│──▶│ 03 HOOK  │──▶│04 AUTOMATE│
│ IG分析   │   │ ニッチ調査│   │ 台本生成 │   │ フック   │   │ 日次自動化│
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
```

---

## 00. SETUP — IGアカウント分析

**ファイル:** [`../prompts/00-setup.md`](../prompts/00-setup.md)

現状のInstagramアカウント（または想定アカウント）を分析し、ポジショニング・強み・改善余地を把握する。プロフィール、既存投稿、フォロワー層、競合を洗い出し、以降のフェーズの土台を作る。

- **入力:** アカウントURL / 既存投稿データ / ブランド情報
- **出力:** アカウント分析レポート、ポジショニング仮説、KPI初期値

## 01. NICHE — バイラル分析

**ファイル:** [`../prompts/01-niche.md`](../prompts/01-niche.md)

ペット × 飼い主啓蒙ニッチにおけるバイラルコンテンツを分析し、勝ちパターン（フォーマット・トピック・尺・音源・投稿時間）を抽出する。

- **入力:** 00の分析結果、対象ニッチ、参考バイラル事例
- **出力:** 勝ちパターン一覧、トピックのバックログ、優先順位付きテーマリスト

## 02. SCRIPT — リテンション最適化スクリプト生成

**ファイル:** [`../prompts/02-script.md`](../prompts/02-script.md)

選定テーマについて、視聴維持率（リテンション）を最大化する台本を生成する。ネコネコのキャラクター・トーンを保ちつつ、エビデンスに基づいた内容を構成する。

- **入力:** 01のテーマ、[`character.md`](character.md)、エビデンスソース
- **出力:** シーン割り台本（ナレーション／テロップ／ビジュアル指示／尺）、根拠リンク

## 03. HOOK — フックエンジニアリング

**ファイル:** [`../prompts/03-hook.md`](../prompts/03-hook.md)

冒頭3秒でスクロールを止めるフックを設計・複数案生成し、A/Bテスト用のバリエーションを作る。

- **入力:** 02の台本
- **出力:** フック候補（3〜5案）、テロップ／サムネ案、A/Bテスト計画

## 04. AUTOMATE — 日次自動化ワークフロー

**ファイル:** [`../prompts/04-automate.md`](../prompts/04-automate.md)

01〜03を日次で回し、各プラットフォーム向けに書き出し・スケジュール投稿するまでの自動化を定義する。

- **入力:** 01〜03の成果物、配信スケジュール
- **出力:** プラットフォーム別書き出し（尺・アスペクト比・キャプション・ハッシュタグ）、投稿キュー、実行ログ（`../content/` へ保存）

## マルチプラットフォーム配信

04 の書き出しでは各媒体の仕様に最適化する。

| プラットフォーム | 推奨尺 | アスペクト比 | 備考 |
|-----------------|-------|-------------|------|
| Instagram Reels | 15〜60秒 | 9:16 | ハッシュタグ・カバー画像重視 |
| TikTok | 15〜60秒 | 9:16 | トレンド音源・字幕重視 |
| YouTube Shorts | 〜60秒 | 9:16 | タイトル・検索最適化重視 |
| LINE VOOM | 短尺 | 9:16 | 対応可能な場合。国内リーチ補完 |
