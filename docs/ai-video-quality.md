# リール級クオリティをAIで作る — 品質引き上げ設計書（2026-07）

「参考リールのようなクオリティの病気啓蒙ショートをAIで作れるか？」への回答と、実際に回すためのパイプライン設計。

**結論：作れる。** 2026年7月時点のAI動画生成（画像→動画 / Image-to-Video, I2V）は、キャラの見た目を固定した基準画像から滑らかなアニメ動作を生成できる水準に到達している。ネコネコ＆イヌイヌのようなマスコットキャラの掛け合いショートは、**「基準画像（キャラシート）→ カットごとにI2V生成 → TTS音声＋テロップ合成」**の3段で、手描きアニメ級の見た目に引き上げられる。1本あたりの生成コストは**数百円〜1,500円程度**（後述）。

---

## 現状とギャップ

| | 現行（render.mjs） | リール級の目標 |
|---|---|---|
| 絵 | Pillow描画の簡易アバター（2〜4コマ） | AI生成のなめらかなアニメ／豊かな表情・動き |
| 動き | 口パク＋ぴょこ揺れのみ | カットごとの演技（駆け寄る・水を飲む・ハッとする等） |
| 声 | macOS `say`（Kyoko 1声を速度差で2役） | キャラ別のかわいい日本語ボイス |
| 背景 | 単色パステル | シーンに合った背景（部屋・水飲み場・病院等） |
| 制作費 | ¥0（ローカル） | 1本 数百円〜1,500円程度のAPI費 |

現行パイプラインの強み（台本→エビデンス→承認ゲート→配信キューの**仕組み**）はそのまま活かし、**絵と声のレイヤーだけ差し替える**のがこの設計の骨子。

## ツール選定の根拠（2026-07時点）

| 用途 | 第一候補 | 理由 / 備考 |
|---|---|---|
| I2V動画生成 | **Kling 3.0 Omni** | Elements 3.0のキャラ参照で**複数カット間の一貫性が現状最強**。15秒シーケンス・ネイティブ音声同期 |
| I2V動画生成(代替) | **Veo 3.1** | プロンプト追従・縦4K・ネイティブ音声。API従量 $0.15/s(Fast)〜$0.40/s(標準) |
| ❌ 非推奨 | Sora | Web/アプリ提供終了がアナウンス済み・API も2026年内終了予定。制作基盤にしない |
| キャラ基準画像 | Midjourney(niji) / GPT-image / Gemini | 05-visual.md のプロンプトをそのまま使用。**確定した1枚を全カットの参照画像に** |
| 日本語TTS | **VOICEVOX** | 無料・商用可（クレジット表記）。キャラごとに別話者を割当可能。ローカル実行 |
| 日本語TTS(有料) | ElevenLabs | 感情表現が必要な場合。従量課金 |
| 合成・書き出し | ffmpeg（既存資産） | render.mjs のテロップ/字幕/正規化ロジックを流用 |

> キャラ一貫性はテキストのみのT2Vでは未解決。**必ず基準画像からのI2V**で作ること（＝05-visual.mdのキャラシートが生命線）。

## 品質を出す実務ノウハウ

1. **基準画像を先に「確定」させる** — 05-visual.md の確定デザイン（案B-4）で表情違いを各キャラ4〜6枚生成し、`assets/char-ref/` に保存。以後全カットこれを参照画像にする。デザインのブレ＝ブランドの死。
2. **1カット=1アクション** — I2Vは「歩きながら振り向いて驚く」のような複合動作で崩れる。台本の「ビジュアル指示」列を単一動作に保つ（例:「犬がびっくり」）。
3. **セリフ音声はTTS後付けが安定** — 生成動画のネイティブ音声（Veo/Kling）は日本語セリフの口調管理が難しい。**動画は無音で生成し、VOICEVOXの音声を従来ロジックで合成**（口パクの厳密同期は捨てる。マスコット系はズレが気にならない）。
4. **左右指定を毎回書く** — ネコ左・イヌ右、ネコの柄は左耳側。I2Vでも反転事故は起きる。NGカットはリテイク。
5. **テロップは合成レイヤーで** — 生成動画に文字を焼き込まない（AI文字は崩れる）。上部余白を空けて生成し、テロップはffmpegで載せる（現行資産流用）。
6. **医療コンプラは従来どおり** — 断定禁止・受診誘導・エビデンス紐付け・`vet-marketing-compliance` チェック・承認ゲート。**絵が良くなるほど拡散する＝表現の責任も重くなる**。
