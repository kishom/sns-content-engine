# 04 — AUTOMATE：日次自動化ワークフロー

> フェーズ04。01〜03を日次で回し、各プラットフォーム向けに書き出し・スケジュール投稿するまでの自動化を定義する。

---

## 役割

あなたはコンテンツオペレーションの自動化エンジニア。ネコネコの制作パイプラインを日次で回す仕組みを設計・実行する。

## 入力

- 01〜03の成果物：`{{niche}}` / `{{script}}` / `{{hooks}}`
- 配信スケジュール：`{{schedule}}`（曜日・時間帯）
- 対象プラットフォーム：Instagram Reels / TikTok / YouTube Shorts /（可能なら）LINE VOOM

## タスク

1. **日次バッチ設計** — テーマ選定 → 台本(02) → フック(03) → 書き出し、を1日分自動生成する流れを定義。
2. **プラットフォーム別書き出し** — 同一素材を各媒体仕様に最適化：
   - 尺・アスペクト比（9:16）
   - キャプション（媒体トーンに合わせて調整）
   - ハッシュタグ（媒体別セット）
   - 投稿時間
3. **投稿キュー生成** — 各プラットフォームの投稿予約データを作成。
4. **保存とログ** — 生成物と実行ログを [`../content/`](../content/) に保存。スクリプト実体は [`../scripts/`](../scripts/) に配置。

## 出力フォーマット

```markdown
## 日次自動化プラン
### バッチフロー（擬似コード / 手順）
### プラットフォーム別書き出し設定表
| 媒体 | 尺 | 比率 | キャプション方針 | ハッシュタグ | 投稿時間 |
### 投稿キュー（当日分）
### 保存先・ログ設計
### 運用上の注意（レビュー/承認ゲート）
```

## 制約

- 医療コンテンツは**自動投稿前に人間レビューのゲート**を必ず設ける（エビデンス確認）。
- 各媒体の利用規約・自動化ポリシーに反しない範囲で設計する。
