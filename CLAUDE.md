# CLAUDE.md — ネコネコ SNS Content Engine

Claude Code 用の指示書。このリポジトリで作業するときのガイドライン。

## プロジェクトの目的

猫キャラ「ネコネコ」と犬キャラ「イヌイヌ」の2人組（バディ）を使って、ペット飼い主向けのショート動画を **AI で自動生成し、マルチプラットフォームへ配信** するエンジン。犬猫の「あるある」で共感を掴み、掛け合い（犬がボケ→猫が博識に解説）で、獣医学的根拠（エビデンス）に基づいた啓蒙コンテンツを届ける。

## 大原則

1. **エビデンス必須** — 健康・医療に関する主張には必ず一次情報の根拠を紐づける（PMC / ACVIM / Merck Veterinary Manual 等）。憶測で医療アドバイスを生成しない。断定を避け、最終的には「かかりつけ医への受診」を促す。
2. **キャラクター一貫性** — ネコネコ（のんびり博識なツッコミ役）＆イヌイヌ（元気なボケ役）の性格・掛け合い・トーン（[`docs/character.md`](docs/character.md)）を崩さない。押し付けがましくしない。犬・猫で注意点が違うテーマは差分を見せる。
3. **プラットフォーム最適化** — 9:16 縦型、冒頭3秒のフック重視、尺・キャプション・ハッシュタグを各媒体（IG Reels / TikTok / YouTube Shorts / LINE VOOM）に合わせる。
4. **ワークフロー準拠** — コンテンツ制作は 00→04 のプロンプトパイプライン（[`docs/workflow.md`](docs/workflow.md)）に従う。

## ディレクトリの使い方

- `docs/` — ワークフローとキャラクター設定のドキュメント
- `prompts/` — 各フェーズのプロンプトテンプレート（編集の中心）
- `content/` — 生成された台本・キャプション・動画メタデータの保存先
- `scripts/` — 配信自動化・パイプライン実行用スクリプト

## 作業スタイル

- プラン・確認・質問は最小限に。即実装 → 検証まで一気に通す。
- 医療情報を扱うコンテンツを生成・編集した場合は、根拠リンクが付いているか必ず確認する。
- 出力は日本語（ターゲットが日本のペット飼い主のため）。

## 自動エージェント適用ルール

`.claude/agents/` のエージェントは、以下の場面では**指示がなくても自動で適用する**こと。

| 場面 | 適用エージェント | 見るポイント |
| --- | --- | --- |
| リール台本の作成・改稿 | `short-video-coach` | フック → 30 秒本編 → CTA の構成、冒頭 1〜2 秒の掴み |
| 健康情報を含むコンテンツ | `vet-marketing-compliance` + `content-creator` | 医療表現の妥当性と、キャラクター（ネコネコ／イヌイヌ）らしさの両立 |
| **全投稿**（台本・キャプション・サムネ文言を問わず公開前に必ず） | `vet-marketing-compliance` | 「断定禁止・受診誘導」チェックを通す |

- 1 つの変更が複数の場面に該当する場合は、該当するエージェントを**すべて**適用する。
- `vet-marketing-compliance` のチェックは例外なく全投稿に適用する（省略しない）。
